# Contributing to Awesome Claw

Thank you for your interest in contributing to Awesome Claw! This document provides guidelines for contributing to this curated list of the OpenClaw ecosystem.

## How to Contribute

### Adding a New Entry

1. **Ensure your entry is relevant**: The project should be related to the OpenClaw ecosystem (forks, variants, tools, resources, etc.)

2. **Check for duplicates**: Make sure the project isn't already listed

3. **Follow the format**: Use the existing format in the relevant section:
   - For table entries: `| **Name** | Description | [Link](url) |`
   - For detailed entries: Follow the pipe table format with Name, Repo, Website, etc.

4. **Include key information**:
   - Project name and link
   - Brief description (keep it concise)
   - Relevant metadata (language, stars, license, etc.)
   - Link to sources/references when applicable

5. **Alphabetical order**: Add new entries in alphabetical order within their section

6. **One pull request per addition**: This makes review easier

### Updating Existing Entries

- Fix broken links
- Update outdated information
- Add missing metadata
- Improve descriptions

### Pull Request Guidelines

1. Create a new branch for your changes
2. Make your changes following the format above
3. Include a clear description of what you're adding/updating
4. Verify all links work before submitting
5. Be responsive to feedback

### Quality Criteria

Projects should meet these criteria:

- **Active or noteworthy**: Either actively maintained or significant enough to be historically relevant
- **Documented**: Has a README or documentation explaining what it does
- **Open source** (for code projects): Source code is publicly available
- **Functional**: Actually works and provides value

### What NOT to Add

- Personal projects without significant community adoption
- Commercial products without free/open tier
- Abandoned projects (unless historically significant)
- Duplicate entries
- Projects unrelated to the OpenClaw ecosystem

## Code of Conduct

This project adheres to a standard of respectful and constructive collaboration:

- Be respectful and inclusive
- Provide constructive feedback
- Focus on what's best for the community
- Show empathy towards others

## Questions?

Feel free to open an issue if you're unsure about:
- Whether a project qualifies
- Which section to add it to
- Formatting questions

Thank you for helping make Awesome Claw a comprehensive resource for the OpenClaw community!
